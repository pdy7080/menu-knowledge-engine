## 3. 통합 계획 및 로드맵

### 3.1 Sprint 0 (현재) 통합 계획

**통합 데이터셋**:
1. **식품영양성분DB API** (필수)
   - 설정: `.env`에 data.go.kr API 인증키 추가
   - 캐싱: Redis에 3개월 TTL로 캐싱 (API 호출 최소화)
   - Canonical_menus.nutrition_info JSONB 필드에 저장

2. **서울관광재단 식당운영정보** (선택)
   - 초기 로드: 매월 1회 배치 다운로드
   - shop 테이블의 representative_menu 필드에 활용
   - 한계: 서울만, 대표메뉴만 제공

3. **전국일반음식점표준데이터** (선택)
   - 초기 로드: 분기별 1회 다운로드
   - shop 테이블 구축에 활용
   - 자동화: GitHub Actions로 월별 갱신 스크립트

### 3.2 Sprint 1 (OCR 파이프라인) 활용

- **메뉴판 OCR** → **식품영양성분DB API** 검색

- 매칭 성공률 목표: 70%+

- **Phase 3**: 지역 확대 (부산, 대구 등 API 조사 필요)

## 4. 주요 발견사항 및 제약사항

### 4.1 긍정 발견사항 ✅

1. **영양정보는 완벽하다**
   - 157개 항목의 상세 영양성분 데이터
   - API 제공, 완전 무료, 상업용 가능
   - data.go.kr에서 완벽하게 구축됨

2. **음식점 정보는 풍부하다**
   - 전국 2.1M 음식점 DB
   - 매일 갱신 (2일 전 기준)
   - CSV 다운로드 자동화 가능

3. **서울 메뉴 데이터 존재**
   - 167,659개 음식점의 대표메뉴명
   - 상업용 사용 제약 없음
