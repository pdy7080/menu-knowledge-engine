# AI 다국어 메뉴 변환 서비스: 종합 사업화 리서치 보고서

한국을 찾는 **연간 1,637만 외국인 관광객 중 73.2%가 식도락 관광에 참여**하지만, 서울 식당의 34%가 심각한 메뉴 오역을 보유하고 있다. "곰탕"이 "Bear Tang"으로, "육회"가 "Six Times"로 번역되는 현실은 단순한 해프닝이 아니라 **연간 9.26조 원 규모 관광 수입의 품질을 좌우하는 인프라 공백**이다. AI/OCR 기반 한국 음식 메뉴판 자동 번역과 문화적 맥락 설명을 결합한 전문 서비스는 현재 시장에 명확한 빈자리가 존재하며, 2025년 방한 관광객 **1,873만 명**(역대 최고) 전망 속에서 사업적 타이밍이 최적이다.

---

## 1. 경쟁 환경: "번역은 있지만, 설명은 없다"

### 기존 서비스의 지형도

현재 한국 메뉴 번역 시장은 크게 세 축으로 나뉜다. **정부 공공 서비스**, **스타트업/앱 서비스**, **범용 번역 도구**다.

**한국관광공사의 '푸드트립 인 코리아'**(foodtrip.or.kr)는 **8,500여 가지 음식 메뉴 무료 자동번역 DB**를 제공하며, 식당 운영자가 직접 QR 메뉴판을 제작할 수 있는 도구를 갖추고 있다. 영어·일본어·중국어 3개 언어를 지원하고 완전 무료라는 강점이 있지만, UI/UX가 구식이고 현장 보급률이 극히 낮으며 동남아 언어를 지원하지 않는다. 서울 종로구는 2024~2025년 광장전통시장 88곳에서 시작해 160곳으로 **다국어 QR 메뉴판** 시범사업을 확대하고 있으며, 서울시는 2024년 10월 관광 중심지 식당 400곳에 다국어 디지털 메뉴보드 배포 캠페인을 진행했다.

스타트업 영역에서 가장 주목할 만한 경쟁자는 **플리토(Flitto)**다. AI 번역 + 전문 번역가 검수의 하이브리드 방식으로 QR 메뉴판을 생성하며, 글로벌 1,000만 유저를 보유한 번역 플랫폼 기반이다. 해외에서는 **AnyMenu**(50개 언어, iOS 중심), **Menu Translator App**(식단 필터링 기능), **Waygo**(동아시아 언어 오프라인 번역) 등이 B2C 여행자 시장을 공략 중이다.

**네이버 파파고**는 2022년 한국관광공사·문화체육관광부와 협업해 'Food Info' 탭을 도입, 이미지 번역 시 음식 상세 정보를 제공하는 기능을 추가했다. 방한 외국인의 **48.3%가 파파고를 이용**하고 있어 번역 시장의 지배적 플레이어이지만, 고유 메뉴명 처리와 문화적 설명에는 여전히 한계가 있다.

### 시장 공백이 명확하다

| 미충족 니즈 | 현황 |
|---|---|
| 문화적 맥락 포함 설명 | "시래기국"을 "dried radish leaf soup"으로 번역해도 맛·문화·먹는 방법 전달 불가 |
| 손글씨·비정형 메뉴판 인식 | 한국 로컬 식당 다수가 손글씨/분필보드 사용, 기존 OCR 인식 실패 빈번 |
| 동남아 언어 지원 | 태국·베트남·인도네시아 관광객 급증에도 대부분 영·중·일에 집중 |
| 알레르기·식이제한 정보 | 외국어로 된 알레르기 정보 제공 식당 거의 전무 |
| 가게별 고유 메뉴명 처리 | "할머니뼈해장국" 같은 상호명+메뉴명 복합형은 어떤 번역기도 처리 불가 |

결론적으로 **"AI/OCR 기반 자동 변환 + 문화적 맥락 설명 + 다국어 지원"을 통합한 전문 서비스**는 현재 시장에 존재하지 않는다.

## 2. 기술 구현: OCR부터 문화적 번역까지의 파이프라인

### 한국어 OCR 기술의 현재 수준

한국어 메뉴판 인식에서 **네이버 CLOVA OCR**이 가장 우수한 성능을 보인다. ICDAR 2019 글로벌 챌린지 4개 분야 1위를 차지했으며, 곡선 배열·기울어진 문자·필기체까지 인식하고 읽는 순서와 방향을 추정하는 기능을 갖추었다. **Google Cloud Vision API**는 80개 이상 언어를 지원하며 월 1,000건 무료 후 **$1.50/1,000건**으로 가격 경쟁력이 있다. **Upstage Document AI**는 한국어 문서 특화 성능이 우수하다.

메뉴판의 특수한 과제는 손글씨, 디자인 폰트, 세로 텍스트, 다중 컬럼 레이아웃 등이다. 이를 해결하기 위해 OCR 후 **LLM 기반 후처리 파이프라인**(메뉴명·가격·설명 분리)을 구축하는 것이 권장된다.

### RAG 기반 설명 생성이 최적 해법

메뉴 설명 생성에는 **Fine-tuning보다 RAG(Retrieval Augmented Generation) 접근법**이 우선 권장된다. 한식 DB, 영양 정보, 문화 설명 등을 벡터 DB에 저장하고, LLM이 검색된 정보를 참고해 정확한 설명을 생성하는 방식이다. 구축 비용이 수백만~수천만 원 수준이고, 문서 교체만으로 즉시 데이터를 업데이트할 수 있어 신메뉴 추가가 빈번한 환경에 유리하다.

### 한국 음식 데이터베이스 구축 자원

- **식약처 식품영양성분 DB**: 약 92,000개 식품 영양정보 Open API
- **한식재단 한식 아카이브**: 9,083건 한식·음식문화 콘텐츠
- **AI Hub**: 한식 이미지 데이터 15만 장(150종×1,000장), 400종 음식 이미지
- **한국국제교류재단**: 약 700종 한국음식 영문 정보
- **농촌진흥청**: Korean Food Composition Table

### 권장 기술 아키텍처

```
메뉴판 이미지 → 전처리 → CLOVA OCR → LLM 레이아웃 분석
→ RAG 시스템(한식 DB 검색 + LLM 설명 생성)
→ 다국어 Transcreation → 로마자 표기 → 이미지 매칭 → 디지털 메뉴 출력
```

## 3. 시장 규모: 1,637만 관광객과 46만 외식업체

### 인바운드 관광의 폭발적 회복

- 2024년 방한 외래관광객: **1,637만 명** (2019년 대비 94% 회복)
- 2025년 상반기: **882.6만 명** (역대 최고치 경신, 2019년 동기 대비 +4.6%)
- 2025년 연간 예측: **1,873만 명** (야놀자리서치), K-컬처 지속 시 2,000만 명 돌파 가능
- 국적별: 중국 24%, 일본 20%, 대만 9%(+40.4%), 미국 8%(+43.7%)
- 구조적 변화: 개별여행(FIT) **82.9%**, 30세 이하 35.6%, 방한 계기 1위 한류 콘텐츠(38.3%)
- 관광 지출 중 식음료 비중: **19.2%**

### 외국인의 가장 큰 불편: 언어 장벽

문화체육관광부 조사 외국인 개선 요구 1위: **"완성도 높은 다양한 외국어 지원"**

### 외식업체의 디지털 전환 가속

- 전국 한식 외식업체: **46만 219개소**
- 키오스크 도입 긍정: 자영업자 **71.2%**
- 배달 채널 비중: 2019년 9% → 2024년 **35%**
- 2026년 1월: 15평 이상 매장 배리어프리 키오스크 의무화

### 정부 정책이 순풍

- 농림축산식품부: 세계 한식 시장 **2027년까지 300조 원** 목표
- 소상공인 디지털 전환 지원: 스마트 상점 기술 보급(최대 500만 원)

## 4. 수익 모델: 90% 이상의 마진 구조

### B2B SaaS 가격 체계

| 플랜 | 기능 | 월 가격 |
|---|---|---|
| Free | 메뉴 10개, 1개 언어, 워터마크 | 무료 |
| Basic | 메뉴 50개, 3개 언어 | 1.9~2.9만 원 |
| Pro | 메뉴 무제한, 5개 언어, AI 추천 | 4.9~6.9만 원 |
| Enterprise | 다점포, API 연동, 맞춤 브랜딩 | 9.9~19.9만 원 |

번역 원가: 100개 메뉴 × 5개 언어 = 약 700원 → **마진율 90% 이상**

### B2C 수익화

- 프리미엄 구독: 알레르기 필터, 오프라인 저장, AI 맞춤 추천 (월 $2.99~$4.99)
- 식당 프로모션 광고
- 예약 연계 커미션: 예약당 5~20%

### 한국음식 DB API (장기 핵심 자산)

- 글로벌 Food API 시장 존재하나 **한국 음식 특화 API 전무**
- 잠재 고객: 배달앱, 관광 플랫폼, 식당 예약 앱, 네이버지도, 항공사·호텔 체인

## 5. 사업화 로드맵: 명동 10개 식당에서 시작한다

### Phase 1: 컨시어지 MVP (0~3개월, 3,000~5,000만 원)

- 핵심 기능: 메뉴판 OCR + 한식 DB 문화 설명 번역 + QR코드 웹페이지
- 언어: 영어·일본어·중국어(간체/번체) 4개
- 기술 스택: Next.js + FastAPI + CLOVA OCR + GPT-4o/HyperCLOVA X + PostgreSQL + pgvector
- 전략: "오즈의 마법사" 방식 컨시어지 MVP로 1~2주 내 출시

### 초기 시장 진입

1. **명동** (외국인 방문률 85.9% 1위) → 30~50개 식당 무료 파일럿
2. 홍대 (52.8%, 인스타그램 바이럴)
3. 이태원 (외국인 비율 25%+)
4. 성수동 (외국인 방문 116% 증가)

### 파트너십

- **최우선: 한국관광공사** — 기존 서비스의 AI/OCR 자동화 기술 파트너
- 지자체 관광특구 사업, 소상공인 디지털 전환 지원금, 배달앱 외국인 모드 API

### 타임라인

- 3개월: MVP 완성
- 6개월: 식당 50개 확보, 관광공사 파트너십
- 12개월: API 베타 출시, 식당 1,000개 → 시리즈 A 투자 유치

## 결론: K-food 생태계의 "라스트 마일"을 장악하라

**핵심 경쟁우위는 한국 음식 온톨로지 DB**에 있다. 메뉴명·재료·알레르기·문화적 설명·이미지를 체계적으로 구축한 이 DB는 시간이 지날수록 데이터 해자(moat)가 깊어지며, B2B SaaS와 API를 동일 엔진으로 운영해 효율을 극대화할 수 있다.
