# 사업화 로드맵

## Phase 1: 컨시어지 MVP (0~3개월)

### 예산: 3,000~5,000만 원

### 핵심 개발 기능
1. **메뉴판 사진 OCR 인식**
2. **한식 DB 기반 문화적 설명 번역**
3. **식당별 QR코드 웹페이지 생성**

### 초기 언어 지원
영어 · 일본어 · 중국어(간체/번체) = 4개 언어 → 서울 방문 외국인 **80% 이상** 커버

### 기술 스택
| 레이어 | 기술 |
|---|---|
| 프론트엔드 | Next.js |
| 백엔드 | FastAPI (Python) |
| OCR | CLOVA OCR |
| LLM | GPT-4o + HyperCLOVA X |
| DB/RAG | PostgreSQL + pgvector |

### MVP 전략: "오즈의 마법사" 방식
- OCR 결과를 **수동 검수**하는 컨시어지 MVP
- **1~2주 내 프로토타입 출시** → 시장 반응 검증
- 참고 사례: 배달의민족 (전단지 디지털화로 시작), 카닥 (Axure로 하루 만에 MVP)

## 초기 시장 진입: 외국인 밀집 지역 집중 공략

### 진입 순서
| 순위 | 지역 | 외국인 지표 | 전략 |
|---|---|---|---|
| **1순위** | **명동** | 외국인 방문률 85.9% (1위) | 30~50개 식당 무료 파일럿 |
| 2순위 | 홍대 | 52.8% | 인스타그램 바이럴 유리 |
| 3순위 | 이태원 | 외국인 비율 25%+ | 다국적 식당 밀집 |
| 4순위 | 성수동 | 외국인 방문 116% 증가 | 트렌디 F&B 밀집 |

### 식당 영업 메시지
> **"메뉴판 사진만 찍어주시면 2시간 내 다국어 웹 메뉴 완성합니다"**

- 식당 앞 QR코드 스티커 부착 → 외국인 관광객 즉시 접근
- 무료 파일럿으로 진입 장벽 제거

## 파트너십 전략

### 최우선: 한국관광공사
- 기존 '외국어 메뉴판 제작 서비스'의 **AI/OCR 자동화 기술 파트너** 포지셔닝
- 공공 인프라 + 인증 브랜드 활용
- 관광공사가 이미 보유한 8,500개 메뉴 DB와 연계

### 추가 제휴 채널
| 파트너 | 연계 방식 |
|---|---|
| 지자체 관광특구 | 사업 입찰 참여 |
| 소상공인 디지털 전환 지원금 | 최대 500만 원 연계 |
| 배달앱 | 외국인 모드 API 제공 |

## 법적 고려사항

### 기회
- 일반 음식점 알레르기 표시 의무 없음 → **자발적 제공이 차별적·사회적 가치**
- 국민권익위원회 표시 강화 권고 → 향후 법제화 가능성 (선점 효과)

### 리스크 관리 (필수)
| 항목 | 대응 |
|---|---|
| 알레르기 정보 정확성 | 법적 면책 조항(disclaimer) 필수 삽입 |
| 번역 오류 건강 피해 | 배상책임보험 가입 권장 |
