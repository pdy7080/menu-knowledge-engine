# 05. 확장 전략 및 IP 보호

## 1. 해외 한식당 확장 — 옵션이 아니라 필연

### 한국 내 vs 해외 한식당의 Pain Point 비교

| | 한국 내 식당 | 해외 한식당 |
|---|---|---|
| **메뉴 언어** | 한국어만 있음 | 현지 언어 있음 |
| **Pain Point** | 한국어 → 외국어 번역 필요 | 메뉴명은 있지만 **설명이 없음** |
| **예시** | "곰탕" → "Bear Soup" 오역 | "Galbi-jjim" → "그래서 이게 뭔데?" |
| **필요한 것** | 번역 + 설명 | **설명 + 문화적 맥락** |

### Menu Knowledge Engine이 해외에서 작동하는 방식

```
해외 한식당
 → 메뉴명 입력 (영어/현지어/로마자)
 → DB 개념 매칭 (Galbi-jjim → canon_xxx_갈비찜)
 → 현지 언어 문화 설명 자동 생성

결과:
"Galbi-jjim: Korean braised short ribs,
 slow-cooked in soy sauce with chestnuts and jujubes.
 Sweet and savory. A celebratory dish often served
 at Korean holidays."
```

> **이건 "번역"이 아니라 K-Food 해설 엔진**

### TAM 확장

- 한국 인바운드 관광 → 한국 내 46만 한식 외식업체
- **+ 글로벌 한식 시장** → 해외 한식당 15,000개 (정부 목표)
- **+ 한식에 관심 있는 글로벌 소비자** → K-food 세계화 시장 300조 원 (2027년 목표)

### 해외 진출 우선순위

| 순위 | 시장 | 이유 |
|---|---|---|
| 1 | 일본 | 한식 인지도 높음, 한식당 다수, 언어 DB 이미 구축 |
| 2 | 미국 (LA/NY) | 한식 붐, 설명 니즈 높음, 영어 DB 기본 |
| 3 | 동남아 (태국/베트남) | 한류 영향, 한식당 급증, 아직 설명 인프라 전무 |
| 4 | 유럽 (런던/파리) | 미쉐린 한식 레스토랑 증가, 프리미엄 시장 |

## 2. 주문 도우미 확장 — 메뉴를 넘어선 "한국 식당 문화 가이드"

### 메뉴 이해를 넘어 "주문 상황 전체"를 도와주는 확장

외국인이 한국 식당에서 겪는 혼란은 메뉴만이 아님:

| 상황 | 외국인의 혼란 | 엔진이 제공할 수 있는 답 |
|---|---|---|
| "곱빼기" | 사이즈 업? 추가 메뉴? | "Double portion (+₩1,000~2,000)" |
| "반찬" | 이거 무료야? 추가 주문? | "Free side dishes, refills available" |
| "셀프 코너" | 뭘 직접 가져와? | "Self-service: water, cutlery, side dishes" |
| "1인분 주문 불가" | 왜 혼자 못 먹어? | "Minimum 2 servings required (sharing culture)" |
| "공기밥" | 밥이 따로? | "Steamed rice served separately (+₩1,000)" |
| "음료 무한리필" | 리필 가능? | "Free refills on drinks" |

### 구현 방식: Knowledge Graph에 "문화 노드" 추가

```
Menu Knowledge Graph
 ├── Menu Nodes (음식 개념)
 └── Culture Nodes (식당 문화) ← 신규
      ├── 반찬 문화
      ├── 곱빼기 개념
      ├── 셀프 코너
      ├── 최소 주문량
      ├── 상차림 (정식/백반)
      └── 계산 방식 (선불/후불/테이블 결제)
```

## 3. 사장님을 위한 숨은 가치: "메뉴 컨설팅"

### 단순 번역 도구를 넘어 "외국인 맞춤 메뉴 어드바이저"

엔진이 메뉴를 분석하면서 자동으로 인사이트 제공:

**① 메뉴 접근성 리포트**
```
📊 당신 메뉴의 외국인 접근성 점수: 52/100

⚠️ 개선 제안:
• "할머니뼈해장국" (난이도 5) → 외국인이 이해하기 어려움
• "매운 제육볶음" → 맵기 단계 표시 추가 권장
• 채식 옵션 없음 → 비건 관광객 유실 가능
```

**② 경쟁 식당 비교**
```
📍 이 지역 비슷한 식당들의 인기 메뉴:
• 외국인 조회 1위: 비빔밥 (난이도 2)
• 당신 메뉴에 빠진 유형: 비빔밥 계열
```

**③ 외국인 트렌드**
```
📈 최근 외국인 관심 급상승 메뉴:
• 떡볶이 (+45%)
• 마라탕 (+32%)
• 한우 구이 (+28%)
```

## 4. 보호 가능한 IP 구조 — 4가지 방어선

### IP ①: 메뉴 수식어 분해 알고리즘

- "왕얼큰순두부뼈해장국" → 수식어 사전 + 표준 메뉴 그래프 기반 자동 분해
- 단순 형태소 분석 ❌ → **음식 도메인 특화 분해 로직**
- 수식어 사전(Modifier Dictionary)이 핵심 자산
- **보호 방법:** 알고리즘 특허 출원 가능 (BM 특허)

### IP ②: AI 기반 메뉴 정체성 확정 프로세스

```
OCR → 후보 추출 → 웹 서치 + DB 교차검증
→ 확신도 점수(confidence score) → 정규화(canonicalization)
```

- "AI를 번역이 아닌 지식 확정에 쓴다"는 **프로세스 자체가 차별적**
- **보호 방법:** 프로세스 특허 / 영업비밀

### IP ③: 양방향 데이터 수집 구조

- B2B(식당 등록) + B2C(외국인 촬영) 양쪽에서 DB가 자동 성장
- 사용할수록 AI 비용 ↓, DB ↑ → **자기강화 루프(self-reinforcing loop)**
- **보호 방법:** 네트워크 효과 자체가 방어 (시간 = 해자)

### IP ④: Menu Knowledge Graph 구조 자체

- Concept → Canonical → Variant → Modifier 4계층 그래프
- 한국 음식에 특화된 온톨로지(Ontology)
- **보호 방법:** 데이터 자산 (경쟁자가 따라 만들려면 처음부터 쌓아야 함)

### IP 보호 전략 종합

| IP | 보호 수단 | 방어 강도 | 비용 |
|---|---|---|---|
| 수식어 분해 알고리즘 | BM 특허 | ★★★★ | 중 |
| 정체성 확정 프로세스 | 영업비밀 + 특허 | ★★★ | 중 |
| 양방향 데이터 루프 | 네트워크 효과 (시간) | ★★★★★ | 무 |
| Knowledge Graph 데이터 | 데이터 축적 (시간) | ★★★★★ | 무 |

> **가장 강력한 해자(moat)는 특허가 아니라 "시간".**  
> 먼저 시작해서 데이터를 쌓는 것이 최고의 방어.

## 5. 장기 비전: K-Food Intelligence Platform

### 최종 포지셔닝

```
현재 (Year 1):
  "한국 식당 메뉴를 외국인에게 설명해주는 서비스"

중기 (Year 2~3):
  "한국 음식의 모든 것을 구조화한 지식 API"

장기 (Year 3+):
  "글로벌 음식 문화를 연결하는 Knowledge Platform"
  (한식 → 일식 → 중식 → 동남아 → 전 세계)
```

### 궁극적 가치

이 플랫폼이 완성되면:
- 배달앱이 외국인 모드를 만들 때 우리 API를 씀
- 관광 플랫폼이 식당 설명을 할 때 우리 DB를 참조함
- 정부가 한식 세계화 정책을 실행할 때 우리 데이터를 씀
- LLM 회사가 음식 도메인 학습할 때 우리 데이터를 구매함

> **"한국 음식을 이해하려면 이 플랫폼을 거쳐야 한다"**  
> 이것이 K-Food 생태계의 "라스트 마일"을 장악한다는 의미.
