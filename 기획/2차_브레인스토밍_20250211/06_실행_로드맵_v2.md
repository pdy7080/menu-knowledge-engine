# 06. 실행 로드맵 v2

> **1차 조사 로드맵에 Menu Knowledge Engine 전략을 반영한 업데이트 버전**

---

## Phase 0: 기반 구축 (0~1개월) — "엔진의 뼈대"

### 목표
- Menu Knowledge Graph 스키마 확정
- 핵심 메뉴 100개 개념 객체 시범 생성
- 성수야 메뉴 업로드 → 자동 인식 파이프라인 PoC

### 세부 작업

**① DB 스키마 설계**
- 4계층 노드 구조 (Concept → Canonical → Variant → Modifier)
- 관계(Edge) 타입 정의
- PostgreSQL + pgvector 기반 설계

**② AI 프롬프트 설계 (2종 분리)**
- **메뉴 정체성 확정용 프롬프트** (Identity Discovery) ← 핵심
  - "이 메뉴가 무엇인지" 탐색하는 프롬프트
  - 웹 서치 결과를 종합해 개념을 확정
- **설명 생성용 프롬프트** (Translation & Explanation)
  - 확정된 개념을 다국어로 설명하는 프롬프트

**③ 핵심 메뉴 100개 선정 및 개념 객체 생성**

선정 기준:
- 외국인 관광객 빈번 조우 메뉴 (비빔밥, 김치찌개, 삼겹살 등)
- 직역 함정이 있는 메뉴 (곰탕, 육회, 순대 등)
- 합성어/수식어가 많은 메뉴 (뼈해장국 계열, 불고기 변형 등)
- 문화적 설명이 필수인 메뉴 (보쌈, 족발, 막걸리 등)

**④ 수식어 사전 v1 (50~100개)**

우선 수집 대상:
- 맛 수식어: 얼큰, 매운, 순, 달콤, 담백
- 크기/등급: 왕, 특, 대, 소, 곱빼기
- 감성/브랜드: 할머니, 옛날, 시골, 궁중, 원조
- 재료: 한우, 생, 모듬, 해물
- 조리법: 불, 숯불, 직화, 수제

### 산출물
- Menu Knowledge Graph 스키마 문서
- 100개 메뉴 개념 객체 (JSON)
- 수식어 사전 v1
- AI 프롬프트 템플릿 2종
- PoC 데모 (메뉴 사진 → 구조화 결과)

### 예산: 500~1,000만 원
- OCR API 비용 (CLOVA OCR)
- LLM API 비용 (GPT-4o, 100개 메뉴 초기 탐색)
- 개발 인력 (1~2명)

## Phase 1: 데이터 증명 (1~3개월) — "이건 된다"

### 목표
- 명동/홍대 현장 메뉴판 200~300장 수집
- B2C 역방향 MVP 출시
- 메뉴 그래프 v1 (변형 관계 포함)
- 자동 분해 성공률 70% 이상 달성

### 데이터 이중 트랙 실행

```
[트랙 A] 성수야 자동 수집
  - 성수야 가게 등록 시 메뉴 사진 자동 처리
  - 목표: 50~100개 가게 메뉴

[트랙 B] 관광 핵심지 현장 수집
  - 명동 / 종로 / 홍대 / 이태원
  - 현장 촬영 기반 (직접 사진)
  - 목표: 200~300장 메뉴판 사진
  - 선정 기준: 한글 전용 메뉴, 로컬 한식, 노포
```

### B2C 역방향 MVP

```
기능:
  - 웹앱 (PWA, 설치 불필요)
  - 카메라로 메뉴판 촬영
  - OCR → DB 매칭 → 즉시 설명
  - 미등록 메뉴: "Analyzing..." → AI 탐색 → 표시
  - 미등록 식당: "사장님께 추천" 공유 기능

기술:
  - Next.js PWA
  - 카메라 API (MediaDevices)
  - Menu Knowledge Engine API 연동
```

### 검증 지표 (KPI)

| 지표 | 목표 |
|---|---|
| DB 등록 Canonical 메뉴 수 | 500개 |
| 수식어 자동 분해 성공률 | 70%+ |
| B2C 일일 스캔 수 | 50건+ |
| AI 호출 비율 (전체 대비) | 30% 이하 |
| 사장님 만족도 (수정률) | 수정 20% 이하 |

### 산출물
- Menu Knowledge Graph v1 (500+ 메뉴, 관계 포함)
- B2C 웹앱 MVP
- 성수야 통합 프로토타입
- 현장 수집 데이터 200~300장 처리 완료
- 자동 분해 성공률 리포트

### 예산: 3,000~5,000만 원

## Phase 2: 플랫폼 전환 (3~6개월) — "플랫폼으로 인식되기 시작"

### 목표
- 성수야 정식 통합 + QR 메뉴 자동 생성
- API 베타 출시 (외부 플랫폼용)
- 등록 식당 200~500개
- 한국관광공사 기술 파트너십 추진
- 해외 한식당 파일럿 1~2곳

### 주요 작업

**① 성수야 정식 통합**
- 가게 등록 시 다국어 메뉴 자동 생성 (정식 기능)
- QR 코드 자동 발급
- 사장님 대시보드: 메뉴 접근성 점수, 외국인 조회 통계

**② API 베타 출시**
- 외부 개발자/플랫폼 대상 API 공개
- RapidAPI 마켓플레이스 등록
- API 문서화 (Swagger/OpenAPI)
- 가격: 월 $29~$299 (요청 건수 기반)

**③ 파트너십**
- 한국관광공사: 기존 '푸드트립' 서비스의 AI/OCR 자동화 파트너
- 서울시/제주도: 다국어 메뉴판 사업 수행 업체 입찰
- 캐치테이블: 가맹점 메뉴 다국어 변환 연동

**④ 해외 한식당 파일럿**
- 대상: LA 또는 도쿄 한식당 1~2곳
- 방식: 메뉴명 입력 → 현지 언어 문화 설명 자동 생성
- 목적: 해외 시장 PMF(Product-Market Fit) 검증

### 검증 지표 (KPI)

| 지표 | 목표 |
|---|---|
| DB 등록 Canonical 메뉴 수 | 3,000개 |
| 등록 식당 수 | 200~500개 |
| B2C 월간 스캔 수 | 5,000건+ |
| API 외부 호출 수 | 월 1,000건+ |
| AI 호출 비율 | 10% 이하 |
| 유료 전환 식당 | 50개+ |

### 예산: 5,000만~1억 원

## Phase 3: 스케일업 (6~12개월) — "독립 플랫폼"

### 목표
- 등록 식당 1,000개+
- API 정식 출시
- 시리즈 A 투자 유치
- 일본/동남아 시장 진출 준비

### 주요 작업

- Menu Knowledge Engine 독립 브랜딩
- API 정식 상용화 (SLA, 프리미엄 플랜)
- 배달앱/관광 플랫폼 B2B 영업
- 지자체 사업 수주 (서울 외 부산, 제주, 인천 등)
- 해외 한식당 50~100개 확보
- 데이터 분석 리포트 상품화

### 투자 유치 스토리

```
"우리는 한국 음식의 모든 것을 구조화한 지식 플랫폼입니다.

• Menu Knowledge Graph: 3,000+ 메뉴 개념, 10,000+ 변형
• 양방향 데이터 루프: B2B + B2C에서 자동 성장
• AI 비용 곡선: 시간이 갈수록 마진 증가
• API 수익: 배달앱, 관광 플랫폼, 호텔이 이미 사용 중
• 해외 확장: 한식 → 일식 → 중식 → 글로벌"
```

## 리스크와 대응

| 리스크 | 확률 | 대응 |
|---|---|---|
| OCR 정확도 부족 (손글씨) | 높음 | LLM 후처리 + 수동 검수 병행 |
| 식당 사장님 무관심 | 중 | 무료 제공 + 즉각적 가치 전달 (2시간 내 완성) |
| 파파고/구글이 유사 기능 출시 | 중 | Knowledge Graph 데이터 해자가 방어 |
| 번역 오류로 인한 알레르기 사고 | 낮지만 치명적 | 면책 조항 + 배상책임보험 + "참고용" 명시 |
| 데이터 편향 (성수동) | 중 | 이중 트랙 수집 (트랙 B: 관광 핵심지) |
