## 이 문서가 존재하는 이유

2차 브레인스토밍에서 도출된 모든 개념 — Knowledge Graph, 수식어 분해, 역방향 B2C,
난이도 스코어, 문화 노드, 성수야 통합 — 은 전부 하나의 질문으로 수렴한다:

> **"이걸 어떤 데이터 구조로 담을 것인가?"**

이 설계가 확정되면:
- MVP UX는 그 위에 얹기만 하면 됨
- API 엔드포인트는 자동으로 정의됨
- AI 프롬프트는 출력 포맷이 확정됨
- 특허 문서는 이 구조를 그대로 옮기면 됨
- 팀/AI/외주에게 같은 언어로 설명 가능

## 설계 원칙 (이 폴더의 모든 문서에 적용)

1. **그래프처럼 사고하되, RDB로 구현한다** — PostgreSQL + relation table
2. **논리 모델이 물리 모델보다 먼저** — 개념이 확정된 후 테이블을 짠다
3. **MVP에 필요한 것만 v0.1에 넣는다** — 확장은 v0.2+에서
4. **AI 출력 포맷 = DB 입력 포맷** — 프롬프트 설계와 스키마가 정합해야 한다
5. **모든 노드에 출처(evidence)를 기록한다** — 신뢰도 추적 가능해야 한다
